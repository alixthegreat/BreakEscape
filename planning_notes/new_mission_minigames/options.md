# New Minigame Options — Missions 2–8

**Purpose:** Identify reusable interactive minigames that would enrich missions M2–M8 and the broader campaign, adding physical security attack techniques and environmental interaction variety beyond what already exists.

**Existing minigame catalogue (not duplicated here):** PIN safe, lockpick, RFID/cloner, password terminal, CyberChef, VM launcher/flag station, network segmentation map, SIEM dashboard, alarm panel, VPN log analyser, infusion pump, dual-auth panel, backup recovery console, EHR terminal, ESD pushbutton, SIS threshold panel, incident command board.

---

## Proposed Minigames

| ID | Name | Short Description | Primary Mission(s) | Security Theme | Gameplay Value |
|----|------|-------------------|-------------------|----------------|----------------|
| MG-A | **Mag Stripe Skimmer** | Handheld device reads magnetic stripe data from a physical card (hotel card, access card, payment card). Player must position skimmer against the card prop, then replay the raw track data to clone or forge access. | M3 (Zero Day HQ), M5 (Quantum Dynamics), M6 (HashChain Exchange) | Physical credential theft; mag stripe vs chip comparison; Track 1/2 data structure | Teaches a physically grounded attack; contrasts with RFID cloner (EM4100 vs MS) to show breadth of card attack surface |
| MG-B | **Shredded Document Reconstruction** | Strips of a shredded document are presented as draggable tiles. Player arranges them into a readable page to reveal a password, PIN, insider contact name, or policy detail. Difficulty scales with strip count and rotation. | M3 (Zero Day shredder), M5 (Quantum Dynamics recycling), M8 (SAFETYNET archives) | Paper-shredder OSINT; physical disposal policy; dumpster diving | Classic escape-room puzzle adapted to teach information destruction policy; highly replayable through randomised strip order |
| MG-C | **Combination Padlock** | Rotary combination dial (L–R–L click sequence). Player must locate the three-number combination from environmental clues (graffiti, date on notice, worn digit markings). Supports directional notation (e.g. L28–R13–L7). | M2 (hospital equipment store), M4 (SCADA facility outbuilding), M6 (server cage) | Physical access control; combination hygiene; security-by-obscurity limits | Teaches that physical locks are only as strong as combination secrecy; contrast with keyed locks already in game |
| MG-D | **Wax Key Impression** | Player obtains a key mould (bar of soap, wax tablet, or clay prop). Interacting with a borrowed/stolen key creates an impression. A second step "cuts" the wax key and tests it in a lock. Fails if mould was taken from a worn-down key. | M4 (facility master key), M5 (server room closet), M8 (interrogation room) | Key impression attack; physical key security; key control policy | Tactile physical-security attack rarely depicted in digital games; teaches key auditing importance |
| MG-E | **Biometric Spoof — Fingerprint Lift** | Two-step puzzle: (1) dust a surface with forensic powder to reveal a latent print, then lift it onto acetate; (2) "mould" the print onto a gelatin finger and present it to a fingerprint reader. The reader has a configurable liveness detection difficulty. | M5 (Quantum Dynamics biometric door), M3 (Zero Day Syndicate executive suite) | Biometric spoofing; liveness detection; something-you-are vs something-you-have | Expands on the existing `fingerprint_kit` item into a full challenge; teaches why biometrics alone are insufficient |
| MG-F | **SDR / RF Scanner** | Software-defined radio UI (waterfall display, frequency slider). Player scans frequency bands to locate a hidden transmitter (bug, wireless repeater, or rogue access point). Correct frequency band + squelch reveals intercepted audio or encoded signal. | M4 (water treatment, find rogue SCADA radio), M7 (SAFETYNET EOC, locate ENTROPY surveillance device) | RF surveillance; rogue device detection; wireless attack surface | Introduces RF as an attack vector; teaches spectrum monitoring; visually distinctive from all existing minigames |
| MG-G | **Bluetooth Pairing Attack** | BLE scanner shows nearby devices and their UUIDs. Player selects a target device (smart lock, medical wearable, or printer) and attempts default-PIN pairing (e.g. 0000, 1234) or replay of a captured pairing handshake stored in a text file clue. | M3 (Zero Day boardroom speaker/lock), M5 (Quantum Dynamics smart cabinet) | Bluetooth default credentials; BLE pairing vulnerability; wireless IoT attack surface | Complements the existing `bluetooth_scanner` item with an interactive challenge rather than passive inventory trigger |
| MG-H | **USB Drop / Autorun Analyser** | Player finds a USB drive. A simulated file browser shows the drive contents (autorun.inf, PDF lure, hidden .exe). Player decides: plug in and run, plug in with sandbox analysis, or submit to digital forensics. Wrong choices trigger a malware event (global variable). | M4 (SCADA facility car park USB drop), M5 (Quantum Dynamics break room), M8 (SAFETYNET locker room) | USB attack vector; autorun policy; removable media controls; social engineering lure | Teaches USB threat model through a consequential decision, not just item pickup |
| MG-I | **Timeline Reconstruction Board** | Evidence cards (log entries, CCTV timestamps, access records, NPC statements) are pinned to a cork board. Player drags cards into chronological order and draws connection lines between related events. Correctly ordered board unlocks the next step. | M5 (Insider Trading evidence correlation), M8 (Mole investigation) | Incident timeline; forensic reconstruction; log correlation | Teaches investigation methodology; reusable for any scenario built around discovering what happened |
| MG-J | **Blockchain Transaction Explorer** | Simplified blockchain explorer showing wallet addresses, transaction hashes, and amounts. Player traces a payment chain across wallet hops, identifies mixing services (indicated by high fan-out), and pinpoints the destination wallet address that links to a known ENTROPY identity. | M6 (Follow the Money — HashChain Exchange) | Cryptocurrency forensics; blockchain traceability; mixer/tumbler evasion; pseudonymity vs anonymity | Directly teaches the M6 CyBOK objective (applied cryptography, financial forensics); visually maps the M2→M5→M6 payment trail |
| MG-K | **Password Hash Cracker Terminal** | John the Ripper-style terminal with wordlist, rules, and hash type selectors. Player observes cracking progress against a hash extracted from the VM. Key choices: wordlist selection (generic vs targeted), rule application (leetspeak, year suffix). Correct crack reveals a credential used elsewhere in the scenario. | M6 (HashChain server room), M8 (SAFETYNET GitList credential) | Password hashing; offline cracking; credential reuse; wordlist relevance | Hands-on representation of the M6 and M8 VM SecGen objectives; teaches why hash algorithm choice and salting matter |
| MG-L | **Steganography Decoder** | Image viewer with histogram, LSB bit-plane viewer, and metadata inspector. Player selects the correct extraction technique (LSB, metadata, colour channel) to reveal a hidden message (key, coordinates, passphrase) embedded in an image found in the scenario. | M3 (Zero Day marketing images), M9 (Digital Archaeology — ENTROPY dead drops) | Steganography; covert channels; digital forensics; carrier analysis | Teaches a technique rarely covered in other minigames; image-based rather than text-based, provides good visual variety |
| MG-M | **Cipher Wheel / Substitution Solver** | Physical cipher wheel with two rotating alphabet rings. Player aligns the wheel according to a key clue (e.g. "A=N") and decodes an intercepted message. A harder variant provides a monoalphabetic ciphertext and frequency analysis bars to solve by hand. | M1 (extension), M9 (Digital Archaeology), M10 (Final Cipher) | Classical cryptography; substitution ciphers; frequency analysis | Complements CyberChef for non-computer-mediated crypto; teaches why classical ciphers fail; low-tech feel appropriate for hidden ENTROPY lore drops |
| MG-N | **File Carver / Hex Viewer** | Disk image browser showing raw sectors. Player uses file signature (magic bytes) matching to carve deleted files from unallocated space. Recovered files (a photo, a document, a credential store) contain mission-critical evidence. | M8 (SAFETYNET GitList server image), M9 (abandoned ENTROPY server) | Digital forensics; file carving; metadata recovery; deletion vs. wiping | Teaches that deletion does not equal destruction; complements the M8 git-history exploitation theme |
| MG-O | **Patch Panel / Cable Routing Puzzle** | Network patch panel with labelled ports and a wiring diagram. Player must reconnect correct cables to re-establish or isolate network segments (or sabotage connections). Wrong patching triggers an alarm or propagates an attack. | M4 (water treatment network re-routing), M7 (SAFETYNET EOC crisis terminal) | Network physical layer; patch panel access control; cable-plant security | Teaches that network security extends to physical cabling; mirrors network segmentation minigame at the physical layer |
| MG-P | **Social Engineering Phone Spoof** | Softphone UI showing caller ID, a preloaded script, and a set of NPC responses. Player selects the correct pretext, caller ID to spoof, and responses to questions (e.g. "Which department are you from?"). Success grants a verbal password or access confirmation over the call. | M2 (impersonate IT to obtain server room code), M5 (vishing attack on Quantum HR), M8 (impersonate director to bypass check-in) | Vishing; pretexting; caller ID spoofing; social engineering indicators | Voice-based social engineering is unique in the game's current repertoire; teaches the same manipulation principles as NPC dialogue but from the attacker's seat |
| MG-Q | **OSINT Profile Builder** | A dossier interface where player fills in a target's profile from clues found across the scenario (LinkedIn card, email footer, voicemail greeting, ID badge photo). Completed profile generates a targeted wordlist or a social engineering script. | M3 (profile Victoria Sterling), M5 (profile Torres to understand motive), M8 (profile each suspect) | OSINT methodology; target reconnaissance; passive information gathering | Ties environmental clue-gathering to a structured output; teaches how passive recon enables active attacks |
| MG-R | **Tamper-Evident Seal Check** | Close-up inspection panel for a tampered device or document. Player compares reference photos of intact seals/labels to the current device state, marking which seals are broken, which have been re-stuck, and which show evidence of heat application. | M4 (SCADA PLC physical tamper), M7 (SAFETYNET equipment locker) | Physical integrity checking; tamper-evident controls; supply chain hardware attacks | Introduces supply chain / hardware tampering as a theme; very low-tech visually, high narrative impact |
| MG-S | **Malware Config Decoder** | A hex dump or partially-obfuscated config file for the ENTROPY malware strain found in the scenario. Player uses a combination of XOR key guess, base64 stripping, and pattern matching to decode C2 server address, activation trigger, and payload parameters. | M2 (hospital ransomware config), M4 (SCADA wiper config), M8 (ENTROPY implant on SAFETYNET server) | Malware analysis; static analysis; obfuscation techniques; IoC extraction | Bridges VM-side forensics to in-game narrative; teaches that malware configuration reveals attacker intent and infrastructure |
| MG-T | **Duress Code / Anti-Coercion Panel** | A keypad that accepts two valid codes: a normal code and a duress code. Player must determine *which* code to enter based on contextual clues (is an NPC watching? Is the NPC coercing them?). Entering the duress code silently alerts a third party. | M5 (insider suspect coercing access), M7 (ENTROPY operative watching terminal entry), M8 (test whether an agent is under duress) | Duress codes; coercion attacks; silent alarms; insider threat detection | Unique game mechanic — the correct answer depends on social context, not just code knowledge; teaches duress protocol design |
| MG-U | **Cryptex / Scroll-Wheel Password Entry** | A row of cylindrical scroll-wheel rings, each displaying a character from a configurable alphabet (letters, digits, or mixed). Player clicks up/down arrows — or scrolls — on each ring to cycle its character. When all rings spell the correct word or code, the cryptex unlocks with a satisfying click-and-open animation. Ring count and alphabet are scenario-configurable (e.g. 5-letter word, 4-digit code, 6-char alphanumeric). Unlike the PIN keypad, the answer is visible as a word across the rings; unlike the password terminal, there is no free-text keyboard — only the wheel interface. Supports a hint strip beneath the rings showing partial clue characters, and an optional worn-ring visual (certain characters appear more faded from frequent use, leaking information about the answer). | M2 (hospital encrypted locker), M3 (Zero Day Syndicate evidence case), M6 (HashChain safe), M9 (ENTROPY archive crate) | Physical password hygiene; password composition; information leakage from physical wear; something-you-know via physical form | Visually and mechanically distinct from both the PIN keypad and the password text field — players encounter the same concept (a secret combination) in a third physical form, reinforcing that authentication weaknesses transfer across formats; the worn-ring variant teaches side-channel information leakage at a physical level |

## Notes on Reuse and Implementation Priority

### Highest narrative impact for M2–M8

| Minigame | Why it fits the upcoming missions |
|----------|-----------------------------------|
| MG-A Mag Stripe Skimmer | M3 and M5 both involve credential theft from corporate environments; adds physical-layer contrast to the RFID cloner already built |
| MG-B Shredded Document | M5 (Quantum Dynamics recycling), M8 (SAFETYNET archive room) both have document-heavy investigation; straightforward to implement as drag-tile puzzle |
| MG-I Timeline Board | M5 and M8 are both investigation/forensics missions; a dedicated reconstruction board would unify evidence gathering into a satisfying culminating step |
| MG-J Blockchain Explorer | M6's entire premise is financial forensics; this is the core interactive element of that mission |
| MG-K Hash Cracker | M6 and M8 both have VM SecGen scenarios built around password cracking — this in-game representation gives a visual counterpart to the real VM work |
| MG-P Phone Spoof | No current voice-based social engineering minigame exists; fits M2 (impersonate IT support) and M5 naturally |

### Easiest to implement (low asset complexity)

MG-B (Shredded Document), MG-C (Combination Padlock), MG-I (Timeline Board), MG-M (Cipher Wheel), MG-T (Duress Code Panel), MG-U (Cryptex Scroll-Wheel) — all involve 2D UI with no 3D or sprite-intensive elements.

### Require custom visual assets

MG-F (SDR waterfall display), MG-A (skimmer device UI), MG-E (fingerprint powder/lift sequence), MG-R (tamper seal close-up) — would need new pixel art panels or animated overlays.

### Cross-scenario reusability rating

| Rating | Minigames |
|--------|-----------|
| High (4+ missions) | MG-B, MG-I, MG-K, MG-P, MG-T, MG-U |
| Medium (2–3 missions) | MG-A, MG-C, MG-D, MG-G, MG-H, MG-L, MG-O, MG-Q, MG-S |
| Niche (1–2 missions, high narrative value) | MG-E, MG-F, MG-J, MG-M, MG-N, MG-R |
